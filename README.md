# AI Daily Brief

Open-source, self-hosted daily news briefing generated by an AI agent and published to GitHub Pages automatically. No server and no always-on computer required.

**Live URL after setup:** `https://<your-username>.github.io/ai-daily`

## What it does

- Runs every morning in GitHub Actions.
- Starts the agent in the cloud.
- Searches and verifies news from the last 24 hours.
- Asks the AI agent for structured JSON content only.
- Renders the final HTML locally with stable styles and layout.
- Publishes the archive to GitHub Pages.
- Shows each article collapsed by default: title + subtitle first, full story after expanding.

## Default coverage

- Developer tools: Claude, Copilot, Cursor, Codex, Windsurf.
- AI tools: non-coding AI products, creative tools, assistants, search, voice, video, image, productivity, and education apps.
- Robotics and drones.
- Defense and autonomous systems.
- Space and astronautics.
- AI startup funding.
- AI market moves only when there is a significant event.

## Cloud setup — recommended

This is the default path. GitHub Actions runs the agent on a schedule and deploys GitHub Pages, so your laptop can be off.

### 1. Fork the repository

Click **Fork** on GitHub. Public repositories can use GitHub Pages and scheduled Actions for free within GitHub's limits.

### 2. Add your Copilot CLI token

The default provider is GitHub Copilot CLI running in GitHub Actions. Create a fine-grained personal access token from the GitHub account that has an active Copilot subscription:

```text
https://github.com/settings/personal-access-tokens/new?name=AI+Daily+Copilot+CLI&description=Run+Copilot+CLI+from+GitHub+Actions&copilot_requests=write
```

Use **Account permissions -> Copilot requests -> Write**.

Go to **Settings -> Secrets and variables -> Actions -> New repository secret**:

| Name | Value |
|---|---|
| `PERSONAL_ACCESS_TOKEN` | Fine-grained PAT with `Copilot requests: write` |

Optional variables in **Settings -> Secrets and variables -> Actions -> Variables**:

| Name | Default | Description |
|---|---|---|
| `COPILOT_MODEL` | Copilot CLI default | Optional Copilot model override |
| `AGENT_PROMPT_PATH` | `agent.md` | Prompt file to load |
| `AGENT_TIMEOUT_SECONDS` | `1800` | Timeout for Copilot CLI generation |

### 3. Enable GitHub Pages

Go to **Settings -> Pages**:

- Source: **GitHub Actions**

The daily workflow will deploy the site to:

```text
https://<your-username>.github.io/<repo-name>/
```

### 4. Enable Actions

Open the **Actions** tab and enable workflows if GitHub asks. The workflow runs daily at **05:00 UTC**. You can also trigger it from **Actions -> Daily AI Brief -> Run workflow**.

## How publishing is guarded

The scheduled workflow renders a dated report and then commits it back to the repository so GitHub Pages updates automatically.

The commit step is intentionally locked down. The workflow fails if the generation step changes anything except:

```text
index.html
outputs/AI_Daily_YYYY-MM-DD.html
```

Copilot CLI is also granted write access only to a temporary ignored JSON file (`.copilot-output/report.json`). The app, not Copilot, renders and commits the final HTML.

## How generation works

The workflow runs:

```bash
python run.py
```

`run.py` calls Copilot CLI in programmatic mode, passes the full system prompt plus `agent.md`, asks Copilot to write structured JSON to `.copilot-output/report.json`, and then renders the returned JSON into HTML. Keep credentials in GitHub Secrets, not in the repository.

The agent output contract is intentionally small:

```json
{
  "title": "AI Daily",
  "tagline": "Short page intro",
  "articles_reviewed": 24,
  "breaking": "",
  "sections": [
    {
      "id": "dev-tools",
      "title": "Developer Tools",
      "articles": [
        {
          "title": "Headline",
          "subtitle": "Collapsed one-sentence summary.",
          "importance": "important",
          "verified": true,
          "facts": ["Fact 1", "Fact 2"],
          "body": ["Context paragraph"],
          "implications": "Why it matters.",
          "source_name": "Source",
          "source_url": "https://example.com",
          "published_at": "2026-05-16"
        }
      ]
    }
  ]
}
```

## Customize the news agent

Open `agent.md` and change the coverage and search queries. You can ask your coding agent to edit it for you, for example:

```text
Edit this agent so it looks for motorization news, especially motorcycles, and does not look for stocks and robotics.
```

Then review the diff. A typical customization is:

- replace the coverage list,
- replace the default search queries,
- remove sections you do not want,
- keep the filtering and verification rules.

The visual HTML structure lives in `run.py`; keep it unchanged unless you want to redesign the report. This keeps costs lower because the model does not regenerate the same HTML and CSS every day.

## Run locally

```bash
git clone https://github.com/<you>/ai-daily
cd ai-daily
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
export COPILOT_GITHUB_TOKEN=github_pat_...
python run.py
```

The agent returns JSON, then `run.py` saves the rendered report to `outputs/AI_Daily_YYYY-MM-DD.html` and rebuilds `index.html`.

## Configuration

| Env variable | Default | Description |
|---|---|---|
| `PERSONAL_ACCESS_TOKEN` | required for Copilot | Fine-grained PAT with `Copilot requests: write` |
| `COPILOT_MODEL` | Copilot CLI default | Copilot model |
| `AGENT_PROMPT_PATH` | `agent.md` | Custom prompt file |
| `COPILOT_OUTPUT_PATH` | `.copilot-output/report.json` | Temporary JSON file written by Copilot CLI |
| `AGENT_TIMEOUT_SECONDS` | `1800` | Timeout for Copilot CLI generation |

To change the schedule, edit the `cron` line in `.github/workflows/daily.yml`. Cron is UTC.

## Cost and safety

Each Copilot run consumes Copilot premium requests from the account behind `PERSONAL_ACCESS_TOKEN`. The repository renders HTML itself, so repeated CSS/layout tokens are not spent on every run.

Safety rules:

- Store API keys only in GitHub Secrets or local environment variables.
- Do not commit `.env`, API keys, OAuth tokens, or CLI credentials.
- Keep the repository public only if the generated reports should be public.
